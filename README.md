# SolarEclipse

[![Verify against the NASA catalog](https://github.com/djEnterprises/SolarEclipse/actions/workflows/verify.yml/badge.svg)](https://github.com/djEnterprises/SolarEclipse/actions/workflows/verify.yml)

**A ten-body Newtonian solar system that predicts real eclipses — and is never told what an eclipse is.**

The Sun, all eight planets, and the Moon, integrated under nothing but pairwise gravity
(velocity Verlet, fixed 15-minute timestep). Initial conditions are seeded once from
analytic ephemerides (JPL orbital elements + an ELP-2000-style lunar theory); after that
the only physics is `aᵢ = Σ G mⱼ (rⱼ − rᵢ) / |rⱼ − rᵢ|³`. Solar eclipses are **detected,
not scripted** — the code watches the Moon's shadow cone and reports when its axis
crosses Earth.

Zero dependencies. Zero build step. Zero network calls. Plain JavaScript that runs in
any browser and verifies headlessly under Node.

**Live showcase:** <https://www.djenterprises.ai/SolarEclipse>

## Verified, not asserted

The claim above is checked against the published NASA five-millennium catalog
(Espenak/Meeus) by a harness you can run yourself in well under a second:

```
node solar-eclipse-sim/verify_range.js 2024-01-01 5
```

Output of exactly that run (Node 24, this repo):

```
  Catalog eclipses in horizon : 10
  Predicted (matched)         : 10   (0 missed, 0 spurious)
  Correct type                : 10/10
  Timing error vs catalog     : mean 5.5 h, max 9.5 h
  Energy drift (relative)     : 1.20e-11
  Compute time                : 0.0 s for 175,320 steps
```

Every solar eclipse in the five-year window — total, annular, partial — with the correct
type, from raw gravity. The same run is enforced in CI on every push (exit code is
nonzero on any miss or spurious detection), alongside two structural gates: the harness
and showcase copies of the physics must stay byte-identical, and the embed bundle must
be byte-exactly reproducible from source. Contributing (human or agent)? Start with
[`AGENTS.md`](AGENTS.md) — the repo's operating manual and invariants.

## What it honestly can't do

- **Accuracy is bounded.** Timing error grows with distance from the seed epoch; within
  ~5 years it stays under half a day. Over a 10-year single seed, misses appear — run
  `node solar-eclipse-sim/verify_range.js 2021-01-01 10` to watch it degrade. The
  showcase defaults to a near-term horizon and offers an epoch re-sync.
- **No ground paths.** It predicts *that*, *when*, and *what type* — never where the
  shadow lands. Path prediction needs sub-arcsecond lunar theory; this is a 15-minute
  Verlet integrator and says so.
- Every number shown in the showcase is measured from a real run, not marketing.

## Repository layout

```
SolarEclipse/                  ← the web showcase (self-contained, relative paths)
├── index.html                 ← nine-chapter long-read + interactive instrument
├── assets/core/               ← the verified physics: ephemeris.js + physics.js
│                                (intentionally DOM-free — this is the portable core)
├── assets/app/                ← SolarSim canvas engine + page interactions
├── embed/                     ← drop-in ambient background for any website
└── data/eclipses.json         ← verification scorecard, generated by a real sim run

solar-eclipse-sim/             ← the demo/verification harness (Node, no deps)
├── verify_range.js            ← catalog audit — the CI gate
├── export_dataset.js          ← regenerates SolarEclipse/data/eclipses.json
├── build_embed.js             ← bundles core+engine+wrapper → embed/solar-background.js
└── index.html                 ← the original standalone instrument page
```

## Quick start

```
git clone https://github.com/djEnterprises/SolarEclipse.git
cd SolarEclipse

# verify the physics against the NASA catalog (< 1 s)
node solar-eclipse-sim/verify_range.js 2024-01-01 5

# open the showcase (any static server; paths are relative)
python3 -m http.server 8000
# → http://localhost:8000/SolarEclipse/
```

## The ambient background (drop-in)

`SolarEclipse/embed/solar-background.js` puts the live inner solar system behind any
page as one `<script>` tag — no dependencies, pauses off-screen, renders a single
static frame under `prefers-reduced-motion`, and reads its entire palette from CSS
custom properties so it adopts the host page's theme (including light themes via
`--sim-vignette: none`). See [`SolarEclipse/embed/README.md`](SolarEclipse/embed/README.md).
It runs in production behind <https://www.djenterprises.ai>.
